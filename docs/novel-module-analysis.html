📦 Key Data Structures

StoryState
types.ts / orchestrator.ts
The central state object containing all story information including characters, world state, relationships, current chapter, and full story text.
CharacterState
types.ts
Contains character status, stress level, emotions, traits, trauma history, skills, secrets, clues, goals, and mind model (public/private self).
Branch
branch-manager.ts
Represents a narrative branch with story segment, branch point, choice made, choice rationale, state after, evaluation scores, events, and structured state.
DeepenedCharacterProfile
character-deepener.ts
Psychological profile including Big Five traits, attachment style, core fear/desire, defense mechanisms, character arc phase, and relationship dynamics.
Motif
pattern-miner-enhanced.ts
Recurring thematic element with occurrences, evolution history, strength, and decay rate.
VisualPanelSpec
types.ts
Complete visual specification including camera settings, lighting, composition, visual prompt, negative prompt, and ControlNet signals.

⚡ Critical Integration Points

1. Main Generation Loop (orchestrator.ts: runNovelCycle)

1. Apply meta-learning configuration 2. Load/create narrative skeleton 3. Roll chaos event (evolution-rules.ts) 4. Generate chaos event with LLM (evolution-rules.ts) 5. Get proactive guidance (motif-tracker.ts) 6. Deepen character profiles (character-deepener.ts) 7. Parse prompt with LLM (orchestrator.ts) 8. Generate story branches OR direct generation 9. Extract state changes (state-extractor.ts) 10. Update world state 11. Analyze relationships (relationship-analyzer.ts) 12. Analyze factions (faction-detector.ts) 13. Track motifs (motif-tracker.ts) 14. Check end-game conditions (end-game-detection.ts) 15. Save state and collect metrics (observability.ts)

2. State Extraction Pipeline (state-extractor.ts)

1. Evaluate turn outcome (LLM) 2. Extract state changes from story text (LLM) 3. Validate and enhance extraction - Apply fact consistency checks (NEW) - Enrich trauma/skill metadata (NEW) - Apply audit rules 4. Apply updates to state 5. Generate evolution summary with typed key events (NEW)

3. Visual Generation Pipeline

visual-orchestrator.ts ↓ (story segment + character states + narrative context) visual-prompt-engineer.ts ↓ (decide: LLM vs hardcoded based on complexity) visual-translator.ts ↓ (emotion → visuals with psychological profile) ↓ (action → camera with theme awareness) VisualPanelSpec[] → save to file

4. Pattern Mining Pipeline (pattern-miner-enhanced.ts)

Every 5 turns: 1. Extract archetypes from characters (LLM) 2. Extract plot templates (LLM) 3. Extract motifs (LLM) 4. Apply decay to all patterns (NEW: with startup calibration) 5. Save to stores (NEW: generic PatternStore)
