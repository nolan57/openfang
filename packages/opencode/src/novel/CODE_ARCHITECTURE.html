System Overview

Core Architecture Pattern

The Novel Engine follows an Orchestrator Pattern with LLM-driven decision making. The EvolutionOrchestrator coordinates all subsystems, delegating specialized tasks to dedicated modules while maintaining story state consistency.

Step-by-Step Flow

1

Command Parsing

User enters slash command via CLI. command-parser.ts validates input and routes to appropriate handler.

2

State Loading

orchestrator.ts loads existing story state from .opencode/novel/state/story_bible.json. Includes characters, relationships, world state, branch history.

3

Chaos System

evolution-rules.ts rolls 2 dice: impact (1-6) and magnitude (1-6). LLM generates specific narrative event based on these parameters.

4

Story Generation

LLM generates story segment (300-500 Chinese characters). Optional multi-branch generation with evaluation and selection.

5

State Extraction

state-extractor.ts analyzes story text via LLM. Extracts: stress changes, new traumas, new skills, relationship deltas, world events.

6

Character & Relationship Analysis

character-deepener.ts infers Big Five personality, attachment style. relationship-analyzer.ts analyzes dynamics, identifies factions.

7

Visual Panel Generation

visual-orchestrator.ts splits story into beats, generates camera specs, lighting, prompts for image generation.

8

State Persistence

All state saved to .opencode/novel/ directory: story_bible.json, dynamic-patterns.json, panels/, summaries/, reflections/.

Module Reference

Core Modules

Core

orchestrator.ts

EvolutionOrchestrator - Main coordinator. Manages story generation cycle, delegates to specialized modules, maintains story state.

Key Methods:

  • loadState() / saveState()
  • runNovelCycle(prompt)
  • generateBranches()
  • switchBranch(branchId)
LLM

model.ts

AI Model Provider - Acquires language model for text generation. Priority: Agent config → Provider default → First available.

Key Functions:

  • getNovelModel() - Returns {providerID, modelID}
  • getNovelLanguageModel() - Returns AI SDK model instance

State Management

State

state-extractor.ts

StateExtractor - Extracts structured state changes from story text. Validates changes against rules, generates evolution summaries.

Key Methods:

  • extract(storyText, state) - LLM-based extraction
  • applyUpdates(state, updates) - Merge changes
  • extractMindModels() - Theory of Mind
  • generateContextString() - For LLM context
State

types.ts

Constants & Types - Standardized tags for trauma, skills, emotions, goals, etc.

Constants:

  • TRAUMA_TAGS - PTSD_Visual, Physical_Pain, etc.
  • SKILL_CATEGORIES - Mental, Social, Technical, Combat
  • CHARACTER_STATUS - active, injured, captured, etc.
  • EMOTION_TYPES - joy, fear, anger, etc.

Analysis Modules

Analysis

character-deepener.ts

CharacterDeepener - Deep psychological analysis using Big Five, Attachment Theory, Character Arc frameworks.

Output:

  • Big Five Traits (1-10 scale)
  • Attachment Style
  • Core Fear / Core Desire
  • Defense Mechanisms
  • Character Arc Phase
  • Narrative Suggestions
Analysis

relationship-analyzer.ts

RelationshipAnalyzer - Analyzes dynamics between characters, identifies factions, generates narrative hooks.

Output:

  • Dynamic Type (ally/rival/mentor/enemy)
  • Power Balance
  • Tension Level
  • Relationship Stage
  • Faction Detection
Analysis

pattern-miner.ts

PatternMiner - Discovers narrative patterns from story text. Auto-generates skill definitions.

Pattern Types:

  • character_trait
  • world_rule
  • skill
  • plot_device
  • tone
Analysis

thematic-analyst.ts

Thematic Reflection - Periodic literary analysis. Runs every 5 turns.

Analysis Areas:

  • Thematic Consistency (1-10 score)
  • Imagery Evolution
  • Character Arc Progression
  • Narrative Pacing
  • Philosophical Depth

Rules Engine

Rules

evolution-rules.ts

EvolutionRulesEngine - Chaos system and state validation.

Chaos Table:

RollImpactEffect
1NegativeCatastrophic Failure
2-3Negative/NeutralComplications
4-5Neutral/PositiveStandard Flow
6PositiveUnexpected Boon
Rules

dynamic-prompt.ts

Prompt Builder - Dynamic prompt construction with tone/style support.

Features:

  • Variable substitution
  • Tone-aware prompts
  • Style injection

Visual Generation

Visual

visual-orchestrator.ts

Visual Orchestrator - Coordinates visual panel generation. Splits story into beats, manages panel sequence.

Output: .opencode/novel/panels/chapter_XXX_panels.json

Visual

visual-prompt-engineer.ts

Hybrid Engine - LLM + Hardcoded fallback for prompt generation. Fast path for standard scenes, LLM for complex.

Decision Logic:

  • Complex emotions → LLM
  • Rare actions → LLM
  • Continuity needs → LLM
  • Standard scenes → Hardcoded

LLM Call Distribution

pie title LLM Calls per Story Generation Cycle "Story Generation" : 35 "State Extraction" : 20 "Branch Generation" : 15 "Chaos Event" : 10 "Character Analysis" : 8 "Relationship Analysis" : 5 "Visual Prompts" : 5 "Thematic Reflection" : 2

LLM Call Optimization Notes

  • Visual Prompts: Uses hybrid engine - LLM only for complex scenes (10-20% of calls)
  • Thematic Reflection: Runs every 5 turns, not every cycle
  • Branch Generation: Optional, only when useBranches=true
  • Character Analysis: Only for major characters (first 5)
