## Key Integration Points

### 1. `analyzeAndEvolve()` - Core Integration Function

**Location:** `novel/orchestrator.ts:1159`

This is the key function connecting evolve and novel:

```typescript
/**
 * Standalone function to analyze and evolve patterns
 */
export async function analyzeAndEvolve(context: string, currentPatterns: any[] = []): Promise<void> {
  // 1. Call LLM to analyze story patterns
  const languageModel = await getNovelLanguageModel()

  // 2. Extract new narrative patterns
  const prompt = `Analyze this story segment and extract unique patterns...`
  const result = await generateText({ model: languageModel, prompt })

  // 3. Save discovered patterns
  const newPatterns = JSON.parse(result.text)
  await saveDynamicPatterns([...existing, ...newPatterns])

  // 4. Check if skills need to be generated
  await checkAndGenerateSkills(context)
}
```

**Call Chain:**

```
CLI: evolve novel
  ↓
novel.ts:handleEvolve()
  ↓
orchestrator.ts:analyzeAndEvolve()
  ↓
PatternMiner: Analyze story → Extract patterns → Save patterns
```

## CLI Command Flow

### `opencode novel evolve` Command

**File:** `cli/cmd/novel.ts:88-95`

```typescript
async function handleEvolve() {
  console.log("Triggering PatternMiner evolution...")

  // 1. Load existing patterns
  const patterns = await loadDynamicPatterns()

  // 2. Get orchestrator and story state
  const engine = await getOrchestrator()
  const state = engine.getState()

  // 3. Call analyzeAndEvolve to analyze entire story
  await analyzeAndEvolve(state.fullStory || "", patterns)

  console.log("✓ Evolution complete!")
}
```

**Execution Flow:**

```
1. opencode novel evolve
   ↓
2. novel.ts:handleEvolve()
   ↓
3. Load patterns from .opencode/novel/patterns/dynamic-patterns.json
   ↓
4. Get full story from orchestrator state
   ↓
5. analyzeAndEvolve(fullStory, patterns)
   ↓
6. LLM analyzes story → Extracts new patterns
   ↓
7. Save new patterns to dynamic-patterns.json
   ↓
8. checkAndGenerateSkills() → May generate new skills
   ↓
9. ✓ Evolution complete!
```

async function handleStart(args: any) {
  // 1. Load prompt
  let promptContent = "Starting new creative session..."
  if (args.prompt) {
    promptContent = await readFile(args.prompt, "utf-8")
  }

  // 2. Create orchestrator
  const engine = await getOrchestrator()

  // 3. Run self-evolution loop
  for (let i = 0; i < loops; i++) {
    // Analyze patterns between rounds
    if (i > 0) {
      await analyzeAndEvolve(promptContent, await loadDynamicPatterns())
    }

    // Generate story
    const result = await engine.runNovelCycle(promptContent)
  }
}
```

**Execution Flow:**

```

2. Load prompt from prompt.md
   ↓
3. Create EvolutionOrchestrator
   ↓
4. Loop N times:
   a. analyzeAndEvolve() → Extract patterns
   b. runNovelCycle() → Generate story
   c. StateExtractor → Extract state changes
   d. EvolutionRules → Apply rules
   e. Save state to .opencode/novel/state/story_bible.json

## Summary

### Relationship Between Novel and Evolve

| Feature            | Novel Engine            | Evolve System           | Relationship                                |
| ------------------ | ----------------------- | ----------------------- | ------------------------------------------- |
| **Purpose**        | Novel story generation  | General evolution       | Novel is an application domain of Evolve    |
| **Core Components**| EvolutionOrchestrator   | Skill/Prompt/Memory     | Share analyzeAndEvolve                      |
| **LLM Access**     | getNovelLanguageModel() | getNovelLanguageModel() | **Identical**                               |
| **Pattern Mining** | PatternMiner            | PatternMiner            | **Identical**                               |
| **Data Storage**   | story_bible.json        | dynamic-patterns.json   | **Mutual read/write**                       |
| **CLI Commands**   | `opencode novel`        | `opencode evolve`       | Novel is a subcommand                       |

### Answer

**Novel and Evolve are not independent systems, but rather:**

1. **Novel Engine is a specialized application of the Evolve self-evolution framework in the domain of fiction writing**
2. **Both share core components:**
   - `getNovelLanguageModel()` - LLM access
   - `analyzeAndEvolve()` - Pattern analysis
   - `PatternMiner` - Pattern mining
   - `Skill` - Skill generation
3. **Shared data storage:**
   - Novel generated story → Evolve analysis input
   - Evolve generated skills → Novel guidance
   - Jointly maintained pattern library

### How `evolve novel` Works

```
User executes: opencode novel evolve
         ↓
CLI: novel.ts:handleEvolve()
         ↓
Load: .opencode/novel/patterns/dynamic-patterns.json
         ↓
Read: orchestrator.getState().fullStory
         ↓
Call: analyzeAndEvolve(fullStory, patterns)
         ↓
LLM analyzes story → Extracts new patterns → Save to pattern library
         ↓
Check if skills need to be generated → Save to skill library
         ↓
Complete!
```

**This is why Novel and Evolve can seamlessly collaborate!**
